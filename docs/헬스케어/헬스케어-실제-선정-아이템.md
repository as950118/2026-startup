## 1. 선정 결과 (결론)

| 항목 | 내용 |
|------|------|
| **선정 아이템** | **욕창·체압 관리 (Pressure Injury Prevention)** — 침대·휠체어용 압력 분포 센서 + 게이트웨이 + 관리 앱·대시보드 |
| **사업명(가칭)** | (예) 압력관리·욕창 예방 B2B 솔루션 — 웰니스 포지션, 구독·SaaS 결합 |
| **1차 타깃** | 병상 수 5~20개 **소규모 요양원·요양병원** |
| **니치 1개 유지** | PMF 전까지 욕창·체압만 집중. 수면·재활·음성 AI 등은 PMF 후 2번째 유스케이스로 검토 |

**선정일(가정)**: 문서 기준. 실제 사업 시 Phase 0 완료(인터뷰 5~10명) 후 최종 확정 권장.

### 1.1 선정 전 최종 체크 (사업화 계획서 §8.2 반영)

실제 사업 시작 시 아래를 확인한 뒤 Go.

- [ ] 우리 팀이 **기술·도메인**에서 욕창·체압 니치를 6개월 안에 MVP로 만들 수 있는가?
- [ ] **타깃 시설(요양원 5~20병상)** 에 5~10명 인터뷰 가능한 인맥·채널이 있는가?
- [ ] **경쟁**이 이미 심한가? (욕창은 니치 많음, 낙상·스마트옷은 회피 유지)
- [ ] **규제**를 “웰니스”로 유지할 수 있는가? (진단·치료 주장 금지)

## 2. 선정 근거 (왜 이 아이템인가)

### 2.1 사업화 계획서 §8 니치 선택 매트릭스와의 정합성

| 기준 | 욕창·체압 | 비고 |
|------|-----------|------|
| **기술 난이도** | 중 | 기성 압력센서·MCU·게이트웨이로 PoC 가능. 커스텀 PCB는 검증 후 |
| **초기 비용** | 중 | 시제품 500만~2천만, 개발 1천만~3천만, 1억 이하 달성 가능 |
| **규제 부담** | **낮음(웰니스)** | 진단·치료 목적 아님 → 의료기기 회피. 검증 후 전환 검토만 |
| **B2B 접근 용이성** | **높음** | 요양원·요양병원은 욕창 관리 의무·리스크 감소 수요 명확, 담당자(간호·관리) 접촉 경로 존재 |
| **추천 순위** | **1순위** | §8.1 매트릭스에서 1순위로 명시 |

### 2.2 “피할 것”과의 정합성

- **낙상방지센서·스마트 옷**: 진입 안 함. 이미 포화.
- **욕창·체압**: “체위·체압·욕창 위험 예측”에 특화. 낙상·스마트옷과 제품·유스케이스가 다름.
- **차별화**: 같은 “센서”가 아니라 **니치(욕창·체압)·AI 위험도 예측·구독**으로 구분.

### 2.3 소액(1억 이하)·실현 가능성

| 단계 | 목표 비용 | 달성 방법 |
|------|-----------|-----------|
| Phase 0 | 거의 0원 | 인터뷰·가설 문서만 |
| Phase 1 (PoC/MVP) | 2천만~3천만 원 | 시제품 500만~2천만 + 개발 1천만 + 운영 500만. 예비/초기창업패키지로 실질 부담 감소 |
| Phase 2 (PMF) | 3천만~1억 원 | 디딤돌·제품화 올인원팩·스타트업 100 등 지원 활용 |
| **총 1억 이하** | 유지 | 구독 수익·지원금으로 재투자, 불필요한 확장 지연 |

### 2.4 수요·시장 정합성

- **국내**: 고령화·요양 수요 증가, 요양원·요양병원 욕창 관리 의무·품질 이슈.
- **B2B 니치**: 소규모 시설은 예산·인력 제한으로 “저비용·효과 명확·구독” 선호.
- **참고 사례**: 마이다스H&T(유연 압력센서·플랫폼) — 우리는 **소규모 요양원·구독·AI 위험도 예측**으로 포지션 구분.

## 3. 선정 아이템 상세 (실행용 스펙)

### 3.1 제품·서비스 (사업화 계획서 §2.1 정리)

| 구분 | 내용 |
|------|------|
| **제품** | 침대·휠체어용 압력 분포 센서 매트 + 게이트웨이 + 관리 앱·대시보드 |
| **기능** | 체위·체압 실시간 측정, 욕창 위험 부위·체위 유지 시간 분석, AI(규칙 기반 → 이후 ML) 기반 위험도 예측·알림, 주간/월간 리포트 |
| **포지션** | **웰니스·건강관리**로 출시. 진단·치료 목적 주장 금지. 검증 후 의료기기 전환 검토 |

### 3.2 기술·구현 요건 (PoC 단계)

| 영역 | 내용 |
|------|------|
| **하드웨어** | 유연 압력 센서 매트(저항·캐퍼시티브 등), ESP32/라즈베리파이 또는 Jetson Orin Nano(엣지 AI 시). BLE/Wi‑Fi 게이트웨이. |
| **SW** | 압력 맵 시각화, 체위 인식, 위험도 스코어(규칙 기반 → 이후 ML), 알림·리포트 생성. |
| **파트너** | 빌드잇·큐레몬·누코드 — 게이트웨이·BLE·소량 양산·인증. |

### 3.3 수익 모델 (고정)

| 항목 | 예시 |
|------|------|
| **디바이스** | 매트+게이트웨이 1세트 **일회성** 또는 할부. 50만~150만 원/세트(병상당). |
| **구독** | **월/연 구독** — 알림·대시보드·리포트·펌웨어 업데이트. 시설당 10만~30만 원/월 수준. LTV 확대 핵심. |

## 4. 검증·실현 가능성 반복 체크

| 질문 | 검증 방법 | 현재 판단 |
|------|-----------|-----------|
| PoC를 3~6개월 안에 만들 수 있는가? | 기성 보드·센서·OEM 사용 시 가능. 커스텀 PCB·완제품 품질은 검증 후 | **가능** |
| 1인~소규모 팀으로 MVP 유지 가능한가? | 니치 1개·기능 1~2개로 제한 시 가능. No-code/Low-code·클라우드 활용 | **가능** |

### 4.2 비용 실현성

| 단계 | 목표 비용 | 달성 방법 |

| 1. PoC/MVP | 2천만~3천만 원 | 시제품 500만~2천만 + 개발 1천만 + 운영 500만. 예비/초기창업패키지 활용 |
| 2. PMF | 3천만~1억 원 | 디딤돌·제품화 올인원팩·스타트업 100 등 |
| **총 1억 이하** | 유지 | 구독·지원금 재투자, 확장 지연 |

### 4.3 수요·검증 실현성

| 질문 | 검증 방법 |
|------|-----------|
| “이 니치에 돈을 쓸 의향”이 있는가? | Phase 0: 타깃 5~10명 미니 인터뷰. “지금 가장 불편한 것”·“예산 범위” 질문 |
| 1곳이라도 실제로 쓸 수 있는가? | Phase 1: 1곳 파일럿(무상/저가) 4주 이상 사용, 피드백 리스트 5개 이상 |

## 6. 대안 시나리오 (No-Go 시)

Phase 0~1 검증에서 **선정 아이템(욕창·체압)이 No-Go**일 때만 대안 검토. **동시에 2개 니치 진행 금지.**

### 6.1 2순위 대안: 실시간 모니터링 + AI 인사이트

| 항목 | 내용 |
|------|------|
| **제품** | 웨어러블·패치·침대 센서 등에서 수집한 생체·활동 데이터 실시간 분석, 이상 징후·트렌드·악화 위험 예측 후 알림·리포트 |
| **차별화** | “모니터링 디바이스”가 아닌 “AI 인사이트·예방 알림” |
| **전환 조건** | 욕창 니치에서 인터뷰 10명 후 “돈 낼 의향” 0명, 또는 파일럿 2개월 후 사용률 20% 미만 등 |

### 6.2 No-Go 판단 기준 (사업화 계획서 §12 반영)

| 조건 | 판단 | 권장 조치 |
|------|------|-----------|
| 인터뷰 10명 후 “돈 낼 의향” 0명 | 니치/타깃 재검토 | 니치 변경(실시간 모니터링+AI 등) 또는 타깃(시설 규모·유형) 변경 |
| 파일럿 2개월 후 사용률 20% 미만 | 제품·UX 문제 | 기능 축소·설치·알림 개선 또는 1곳 더 파일럿, 또는 대안 니치 검토 |
| 유료 전환 시도 5곳 이상 실패 | 가격·제안·채널 문제 | 단가·패키지·결제 방식·담당자 채널 재설계 |
| Phase 2 12개월 후 유료 2곳 미만 | PMF 미달 | 채널·제품·지역 중 1개 축 변경 또는 사업 범위 축소 |

## 7. 실행 포인트 (첫 3개월)

사업화 계획서 §13과 동일 방향. **선정 아이템 = 욕창·체압** 기준으로 구체화.

### 7.1 Month 1: 니치·가설 확정

| 주 | 할 일 | 산출물 |
|----|-------|--------|
| 1 | **선정 아이템 확정**: 욕창·체압 관리. 가설 1페이지 작성. “소규모 요양원이 욕창 관리 부담·리스크 때문에 인력·품질 압박을 받는다 → 우리는 압력센서+대시보드+알림으로 체위·위험을 가시화해 부담을 줄인다” | 니치 1개, 가설 문서 |
| 2 | 타깃 시설(요양원 5~20병상) 리스트업, 접촉 채널 확보 | 타깃 리스트 20곳 이상 |
| 3~4 | 미니 인터뷰 5~10명. “가장 불편한 것”·“예산·도입 절차” | 인터뷰 노트·인사이트 1페이지 |

### 7.2 Month 2: PoC 범위·자원 확정

| 주 | 할 일 | 산출물 |
|----|-------|--------|
| 1 | 인터뷰 결과 반영해 가설 수정. PoC 기능 1~2개 고정(예: 압력 맵 시각화 + 위험 알림 1종) | 수정 가설·PoC 스펙 |
| 2 | 하드웨어: 기성 압력센서·ESP32/라즈베리파이 선정. SW: 대시보드·알림 1개 목표 | 부품·SW 스택 목록 |
| 3 | 예비/초기창업패키지 공고 확인·신청 준비. 파일럿 후보 1~2곳 접촉 | 지원금 일정·파일럿 후보 1~2곳 |
| 4 | PoC 개발 착수(또는 외주·파트너 일정 확정) | 개발 일정·마일스톤 |

### 7.3 Month 3: PoC 완성·파일럿 계약

| 주 | 할 일 | 산출물 |
|----|-------|--------|
| 1~2 | 시제품 1~5대 완성. 대시보드·알림 1개 동작 확인 | 동작하는 PoC |
| 3 | 파일럿 1곳과 계약(무상·저가). 설치·교육·사용 시작 | 파일럿 계약·사용 시작 |
| 4 | 2주차 사용률·피드백 수집. “다음에 바꿀 점” 5개 목표 | 피드백 리스트 |

사업화 계획서 §11·§3.2 반영. **갭만 공략** — 기존 제품과 겹치지 않는 포지션 유지.

| 분야 | 참고 사례 | 우리와의 차별화 포인트 |

| **욕창·압력** | 마이다스H&T(유연 압력센서·플랫폼) | 니치(소규모 요양원)·구독·AI 위험도 예측·저단가 패키지 |
| **웨어러블·모니터링** | 휴이노(MEMO Cue 등) 패치+앱 | 우리는 욕창·체압 1개 니치·침대/휠체어 매트·B2B 구독 |
| **일반 낙상·스마트옷** | (다수 존재) | **진입 안 함**. 포화. |

## 9. 분기별 검증 체크포인트 (반복 검증)

사업화 계획서 §12 반영. **분기마다** 아래를 점검하고, No-Go 시 §6 대안·피벗 검토.

| 시점 | 질문 | Go 조건(예시) |
|------|------|----------------|
| **Q1** | 니치·가설이 타깃 인터뷰로 지지되는가? | 인터뷰 5명 이상, “불편함·예산” 일치 |
| **Q2** | 1곳 파일럿이 실제로 쓰이고 피드백이 나오는가? | 4주+ 사용, 개선점 5개 이상 |
| **Q3** | 유료 전환이 1곳이라도 되는가? | 1곳 유료 계약 또는 구독 의향서 |
| **Q4** | 2~5곳 유료, 갱신률·단위 경제가 나오는가? | 유료 2곳+, 월 갱신률 90%+ |
| **이후** | 스케일업 1축이 수익·비용 대비 타당한가? | LTV/CAC 2배+, 확장 비용 회수 가능 |
