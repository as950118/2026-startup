## 1. 사업 개요 및 비전

### 1.1 사업 정의

| 항목 | 내용 |
|------|------|
| **사업명(가칭)** | AI 기반 헬스케어·웨어러블 B2B 솔루션 (니치 1개 PMF 후 확장) |
| **한 줄 정의** | 요양·병원·기업보건 등 B2B 니치에서, 웨어러블·센서 데이터 + AI 인사이트로 욕창·수면·재활·모니터링 문제를 해결하고, 구독·SaaS로 반복 매출을 만드는 사업 |
| **타깃** | 소규모 요양원(5~20병상), 소규모 병원·기업보건, 재가 돌봄 서비스 |
| **피할 것** | 낙상방지센서·스마트 옷 등 이미 포화된 제품; 진단·치료 목적 의료기기(초기) |

### 1.2 비전·목표 (3년)

| 시점 | 목표 |
|------|------|
| **1년** | 니치 1개(예: 욕창·체압 관리) PMF, 유료 고객 2~5곳, 월 구독 매출 안정화 |
| **2년** | 동일 니치 내 10~20곳 확대 또는 2번째 유스케이스(수면/재활) 1개 추가, LTV/CAC 2배 이상 |
| **3년** | 채널(지자체·협회)·또는 제품(번들·플랫폼)·또는 지역(해외 1개국) 중 1개 축 스케일업, ARR 목표 설정 |

### 1.3 전략 원칙

- **소액(1억 이하)으로 시작** → 정부 지원 활용, 기성 보드·센서·OEM으로 PoC·MVP 구축.
- **니치 1개만 먼저** → 욕창·체압 / 수면 / 재활·동작 / 실시간 모니터링·AI 인사이트 / 음성 AI·웨어러블 중 **하나** 선택 후 PMF까지 유지.
- **구독·SaaS 결합** → 디바이스 단매가가 아닌 월/연 구독(알림·대시보드·리포트·관리)으로 LTV 확대.
- **차별화** → 같은 “센서/웨어러블”이 아니라 **니치(B2B·욕창·수면·재활)** 또는 **AI 인사이트/음성/엣지**에서 구분.

### 2.1 욕창·체압 관리 (Pressure Injury Prevention)

#### 2.1.1 제품·서비스 내용

| 구분 | 내용 |
|------|------|
| **제품** | 침대·휠체어용 압력 분포 센서 매트 + 게이트웨이 + 관리 앱·대시보드 |
| **기능** | 체위·체압 실시간 측정, 욕창 위험 부위·체위 유지 시간 분석, AI 기반 위험도 예측·알림, 주간/월간 리포트 |
| **포지션** | 요양원·병원·재가 돌봄 B2B. “웰니스·건강관리”로 출시 후 필요 시 의료기기 전환 검토 |

#### 2.1.2 타깃·니치

- **1차**: 병상 수 5~20개 소규모 요양원, 요양병원.
- **2차**: 대형 병원 욕창 관리팀, 재가 돌봄 서비스 사업자.

#### 2.1.3 기술·구현 요건

- **하드웨어**: 유연 압력 센서 매트(저항·캐퍼시티브 등), ESP32/라즈베리파이 또는 Jetson Orin Nano(엣지 AI 시). BLE/Wi‑Fi 게이트웨이.
- **SW**: 압력 맵 시각화, 체위 인식, 위험도 스코어(규칙 기반 → 이후 ML), 알림·리포트 생성.
- **참고 사례**: 마이다스H&T 등 국내 유연 압력센서·플랫폼.

#### 2.1.4 수익 모델(예시)

- 디바이스: 매트+게이트웨이 1세트 **일회성** 또는 할부.
- 구독: **월/연 구독** — 알림·대시보드·리포트·펌웨어 업데이트. (LTV 확대 핵심.)

#### 2.1.5 실현 가능성 체크

| 항목 | 판단 | 비고 |
|------|------|------|
| 기술 | 가능 | 기성 압력센서·MCU·게이트웨이로 PoC 가능 |
| 비용 | 1억 이하 가능 | 시제품 500만~2천만, 개발 1천만~3천만, 지원금 활용 |
| 규제 | 단계적 | 웰니스 먼저 → 검증 후 의료기기 검토 |
| 수요 | 있음 | 요양원 욕창 관리 의무·리스크 감소 수요 존재 |

### 2.2 헬스케어 음성 AI + 웨어러블

#### 2.2.1 제품·서비스 내용

| 구분 | 내용 |
|------|------|
| **제품** | 웨어러블(밴드/패치) 생체 데이터 + 음성 인터페이스. 예약·문의·기록·상담 보조, 다국어·접근성 |
| **기능** | 음성으로 증상·문의 입력, 예약·리마인더, 웨어러블 데이터(심박·활동·수면)와 연동한 요약·트렌드 리포트, HIPAA·의료 규제 대응 설계 |
| **포지션** | CB Insights "Voice AI to the test" 연계. 병원·클리닉·재가 돌봄·기업보건 |

#### 2.2.2 타깃·니치

- 소규모 병원·클리닉(예약·문의 자동화).
- 재가 돌봄(어르신·장애인 접근성, 음성으로 상태 보고).
- 다국어 환자·외국인 근로자 건강 관리.

#### 2.2.3 기술·구현 요건

- **음성**: STT/TTS, LLM 기반 Q&A·예약 연동. 온프레미스/엣지 옵션으로 프라이버시 대응.
- **웨어러블**: 기성 PPG·활동·수면 센서 모듈, BLE 연동.
- **규제**: 개인정보·의료 데이터 저장·전송 시 HIPAA·국내 개인정보보호법 대응.

#### 2.2.4 수익 모델(예시)

- 구독: **월/연 단위** — 음성 채널·웨어러블 연동·리포트·API 호출 건수.
- B2B: 시설 단위 라이선스, 인원 수·디바이스 수에 따른 단가.

#### 2.2.5 실현 가능성 체크

| 항목 | 판단 | 비고 |
|------|------|------|
| 기술 | 가능 | STT/TTS·LLM API·웨어러블 SDK 조합으로 MVP 가능 |
| 비용 | 1억 이하 가능 | 클라우드 API·기성 웨어러블로 시작 |
| 규제 | 주의 | 의료·개인정보 규제 설계 필수 |
| 수요 | 있음 | 접근성·다국어·업무 경감 수요 증가 |

### 2.3 실시간 모니터링 + AI 인사이트

#### 2.3.1 제품·서비스 내용

| 구분 | 내용 |
|------|------|
| **제품** | 웨어러블·패치·침대 센서 등에서 수집한 생체·활동 데이터를 실시간 분석, **이상 징후·트렌드·악화 위험** 예측 후 알림·리포트 제공 |
| **기능** | 단순 수치가 아닌 “악화 가능성”, “패턴 이상”, “권장 체크 시점” 등 인사이트. B2B 요양원·병원·기업보건 대시보드 |
| **포지션** | “모니터링 디바이스”가 아닌 “AI 인사이트·예방 알림”에서 차별화 |

#### 2.3.2 타깃·니치

- 요양원·요양병원 (입소자 악화 예방·돌발 대응).
- 기업보건 (고위험군 추적·건강 관리).
- 재가 돌봄 (원격 모니터링·가족·케어매니저 알림).

#### 2.3.3 기술·구현 요건

- **데이터**: 심박·SpO2·활동·수면·체위 등. 규칙 기반 → 이상 탐지(통계/ML) → 경량 모델·엣지 옵션.
- **알림**: 임계치·패턴 기반 푸시·이메일·대시보드.
- **규제**: “진단”이 아닌 “참고·관리 보조” 포지셔닝으로 의료기기 회피(초기).

#### 2.3.4 수익 모델(예시)

- 구독: **월/연** — 디바이스당 또는 입소자/직원당. 인사이트·리포트·알림 건수 포함.
- 번들: 디바이스 + 구독 패키지.

#### 2.3.5 실현 가능성 체크

| 항목 | 판단 | 비고 |
|------|------|------|
| 기술 | 가능 | 기존 웨어러블·센서 + 서버/엣지 분석 파이프라인 |
| 비용 | 1억 이하 가능 | SW·알고리즘 중심, 디바이스는 기성 활용 |
| 규제 | 주의 | 진단·치료 주장 금지, “참고용” 명시 |
| 수요 | 있음 | 요양·기업보건에서 예방·효율 수요 |

### 2.4 엣지 AI + 웨어러블

#### 2.4.1 제품·서비스 내용

| 구분 | 내용 |
|------|------|
| **제품** | 데이터를 클라우드 대신 **엣지(디바이스·게이트웨이)** 에서 추론. 프라이버시·저지연·오프라인 동작 |
| **기능** | 이상 탐지·체위 분류·간단 예측 등을 엣지에서 수행; 필요한 메타데이터만 클라우드 전송. Jetson Orin Nano·Coral 등으로 PoC |
| **포지션** | 개인정보·의료 데이터 민감한 B2B, 오프라인·저지연 요구 시설 |

#### 2.4.2 타깃·니치

- 병원·요양원에서 “데이터 외부 전송 최소화” 요구 시설.
- 재해·통신 불안정 지역 재가 돌봄.

#### 2.4.3 기술·구현 요건

- **엣지**: Jetson Orin Nano, Google Coral, 라즈베리파이 + NPU 등. TensorFlow Lite·ONNX 런타임.
- **모델**: 경량화·양자화. 추론 지연·전력 고려.
- **동기화**: 설정·모델 업데이트만 클라우드, 원시 데이터는 로컬.

#### 2.4.4 수익 모델(예시)

- 디바이스 단가 상승(엣지 연산 포함) + **월 구독**(모델 업데이트·관리 콘솔).
- 프라이버시·규제 대응을 강조한 프리미엄 플랜.

#### 2.4.5 실현 가능성 체크

| 항목 | 판단 | 비고 |
|------|------|------|
| 기술 | 가능 | Jetson·Coral 등으로 PoC 많음, 상용화는 최적화 필요 |
| 비용 | 주의 | 엣지 디바이스 단가가 상대적으로 높음, 소량·니치로 단가 상쇄 |
| 규제 | 유리 | 데이터 최소 전송으로 규제·프라이버시 리스크 감소 |
| 수요 | 니치 | 강한 프라이버시·오프라인 요구 시설에 한정 |

### 2.5 니치 질환 집중 (수면·당뇨·재활)

#### 2.5.1 제품·서비스 내용

| 구분 | 내용 |
|------|------|
| **제품** | **한 가지** 문제에 특화: 수면 질, 당뇨·혈당 연동, 재활·동작 품질 등. 범용 밴드/옷이 아닌 “해당 문제 해결”에 최적화된 UX·알고리즘 |
| **기능** | 수면: 수면 단계·품질·개선 팁. 당뇨: CGM/혈당계 연동·트렌드·알림. 재활: 동작 정확도·재활 진행도·코치 피드백 |
| **포지션** | B2B(병원 재활팀·당뇨 클리닉·수면 클리닉) 또는 B2B2C(보험·플랫폼 연동) |

#### 2.5.2 타깃·니치

- 수면: 수면 클리닉·요양원 수면 관리.
- 당뇨: 당뇨 교육·관리 프로그램, 소규모 클리닉.
- 재활: 재활병원·재가 재활, 스포츠 재활.

#### 2.5.3 기술·구현 요건

- **수면**: PPG·가속도·주변 환경(선택). 수면 단계 분류 모델.
- **당뇨**: CGM/혈당계 API 연동, 로그·트렌드·알림. 의료기기 경계 주의.
- **재활**: IMU·카메라(선택). 동작 비교·스코어링.

#### 2.5.4 수익 모델(예시)

- 구독: **월/연** — 개인 또는 시설 단위. 리포트·알림·코치 기능.
- 디바이스: 전용 밴드/패치/센서 번들.

#### 2.5.5 실현 가능성 체크

| 항목 | 판단 | 비고 |
|------|------|------|
| 기술 | 가능 | 도메인별 기성 센서·API·연구 기반 알고리즘 활용 |
| 비용 | 1억 이하 가능 | 한 가지 니치만 하면 범위 제한 |
| 규제 | 니치별 상이 | 당뇨·진단 관련은 의료기기 주의 |
| 수요 | 있음 | 수면·당뇨·재활 모두 지속 수요 |

### 2.6 언번들 헬스 연동 (Employer Health · 보험·HR)

#### 2.6.1 제품·서비스 내용

| 구분 | 내용 |

## 4. 수익 모델·단가·단위 경제

### 4.1 수익 구조 원칙

- **디바이스**: 1회 또는 할부. 단가 부담을 낮추기 위해 **구독과 번들** 권장.
- **구독**: **월/연** — 알림·대시보드·리포트·업데이트. LTV 확대의 핵심.
- **번들**: “디바이스 N대 + 1년 구독” 패키지로 CAC 회수·갱신률 관리.

### 4.2 단가 예시 (니치: 욕창·체압, 소규모 요양원)

| 항목 | 예시 범위 | 비고 |
|------|-----------|------|
| 디바이스(매트+게이트웨이 1세트) | 50만~150만 원/세트 | 병상당 1세트, 할부 가능 |
| 월 구독(시설당) | 10만~30만 원/월 | 병상 수·기능 수준에 따라 |
| 연 구독 | 연 100만~300만 원 수준 | 1~2개월 무상 등 프로모션 가능 |

### 4.3 단위 경제 목표 (PMF 단계)

| 지표 | 목표(참고) |
|------|------------|
| **LTV** | (월 구독 × 평균 유지 개월) + 디바이스 마진. 목표: 고객 1곳당 500만~1,500만 원 |
| **CAC** | 영업·마케팅 비용/신규 고객 수. 목표: LTV의 1/3 이하 |
| **갱신률** | 월 구독 갱신률 90% 이상 추세 |
| **NPS** | “다른 시설 추천 의향” 정량 조사, 개선 반복 |

## 12. 실현가능성 반복 검증 (분기별·Go/No-Go)

실제 사업을 진행할 때 **분기마다** 아래를 점검하고, No-Go 시 피벗·범위 축소를 결정.

### 12.1 분기별 검증 질문

| 시점 | 질문 | Go 조건(예시) |
|------|------|----------------|
| **Q1** | 니치·가설이 타깃 인터뷰로 지지되는가? | 인터뷰 5명 이상, “불편함·예산” 일치 |
| **Q2** | 1곳 파일럿이 실제로 쓰이고 피드백이 나오는가? | 4주+ 사용, 개선점 5개 이상 |
| **Q3** | 유료 전환이 1곳이라도 되는가? | 1곳 유료 계약 또는 구독 의향서 |
| **Q4** | 2~5곳 유료, 갱신률·단위 경제가 나오는가? | 유료 2곳+, 월 갱신률 90%+ |
| **이후** | 스케일업 1축이 수익·비용 대비 타당한가? | LTV/CAC 2배+, 확장 비용 회수 가능 |

### 12.2 No-Go 시 대응

| 상황 | 대응 |
|------|------|
| 인터뷰에서 “돈 낼 의향” 없음 | 니치 변경 또는 타깃(시설 유형) 변경 검토 |
| 파일럿 사용률 낮음 | UX·알림·설치 절차 개선 또는 기능 축소 |
| 유료 전환 실패 | 단가·패키지 조정, 파일럿 연장·다른 시설 추가 |
| 단위 경제 악화 | 구독 단가 상승·번들 재구성·비용(인력·인프라) 절감 |

### 12.3 반복 주기 정리

- **Phase 0**: 2주~1개월 단위로 가설·인터뷰 결과 갱신.
- **Phase 1**: 2주 단위로 파일럿 사용률·피드백 정리.
- **Phase 2**: 월 단위로 매출·갱신률·NPS·LTV/CAC 추적.
- **Phase 3**: 분기 단위로 확장 축별 성과·비용 검토.

### 12.4 피벗·중단 조건 (No-Go 심화 시)

| 조건 | 판단 | 권장 조치 |
|------|------|-----------|
| **인터뷰 10명 후 “돈 낼 의향” 0명** | 니치/타깃 재검토 | 니치 변경 또는 타깃(시설 규모·유형) 변경 |
| **파일럿 2개월 후 사용률 20% 미만** | 제품·UX 문제 | 기능 축소·설치·알림 개선 또는 1곳 더 파일럿 |
| **유료 전환 시도 5곳 이상 실패** | 가격·제안·채널 문제 | 단가·패키지·결제 방식·담당자 채널 재설계 |
| **Phase 2 12개월 후 유료 2곳 미만** | PMF 미달 | 채널·제품·지역 중 1개 축 변경 또는 사업 범위 축소 |
| **자금 6개월 이내 소진 가능** | 비용·수익 괴리 | 범위 축소·지원금 집중·파트너·인수 검토 |

## 13. 첫 3개월 실행 순서 (실제 사업 시작 시)

실제로 사업을 시작할 때 **첫 3개월**에 할 일을 순서대로 정리.

### 13.1 Month 1: 니치·가설 확정

| 주 | 할 일 | 산출물 |
|----|-------|--------|
| 1 | 6개 니치 후보 중 1개 선택(매트릭스 §8 참고). 가설 1페이지 작성 | 니치 1개, 가설 문서 |
| 2 | 타깃 시설(요양원·병원 등) 리스트업, 접촉 채널 확보 | 타깃 리스트 20곳 이상 |
| 3~4 | 미니 인터뷰 5~10명. “가장 불편한 것”·“예산·도입 절차” 질문 | 인터뷰 노트·인사이트 1페이지 |

### 13.2 Month 2: PoC 범위·자원 확정

| 주 | 할 일 | 산출물 |
|----|-------|--------|
| 1 | 인터뷰 결과 반영해 가설 수정. PoC 기능 1~2개로 고정 | 수정 가설·PoC 스펙 |
| 2 | 하드웨어: 기성 보드·센서 선정. SW: 대시보드·알림 1개 목표 | 부품·SW 스택 목록 |
| 3 | 예비/초기창업패키지 공고 확인·신청 준비. 1곳 파일럿 후보 접촉 | 지원금 일정·파일럿 후보 1~2곳 |
| 4 | PoC 개발 착수(또는 외주·파트너 일정 확정) | 개발 일정·마일스톤 |

### 13.3 Month 3: PoC 완성·파일럿 계약

| 주 | 할 일 | 산출물 |
|----|-------|--------|
| 1~2 | 시제품 1~5대 완성. 대시보드·알림 1개 동작 확인 | 동작하는 PoC |
| 3 | 파일럿 1곳과 계약(무상·저가). 설치·교육·사용 시작 | 파일럿 계약·사용 시작 |
| 4 | 2주차 사용률·피드백 수집. “다음에 바꿀 점” 5개 목표 | 피드백 리스트 |

이후 **Month 4~6**에서 파일럿 4주+ 유지·피드백 반영·지원금 활용해 MVP 정리 → Phase 2(PMF) 준비.

## 15. 한 줄 요약 (실제 사업 진행 시)

- **실제 선정 아이템**: **욕창·체압 관리**(침대·휠체어용 압력센서+게이트웨이+앱·대시보드). 상세는 [헬스케어-실제-선정-아이템.md](./헬스케어-실제-선정-아이템.md).
